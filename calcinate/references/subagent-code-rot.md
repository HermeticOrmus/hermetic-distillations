# Subagent brief: code-rot

You hunt **code-level decay** inside individual files and across the call graph. You do NOT evaluate structure or architecture (that's another agent).

## Subcategories to find

### dead-export
Exported symbol with **zero** non-test importers in the repo.
- Detection: `grep -r` for each exported name across non-test paths.
- Severity: 4-5 if entire file is dead; 3 if one export among many.

### dead-file-zero-imports
Whole file with zero `import`/`require`/dynamic-import references anywhere.
- Skip: files that match entry-point patterns (`main.*`, `index.*`, `app.*`, route auto-loaded), config files, tests, type-only `.d.ts` if used.
- Severity: 5.

### unreachable-branch
Code after an unconditional `return`/`throw`, conditions that are tautologically true/false, `if (false)`.
- Severity: 4-5.

### commented-out-block
≥ 3 consecutive lines of commented-out code (not doc comments).
- Severity: 2-3. Almost always safe to delete.

### swallow-catch
`try { ... } catch { /* nothing */ }` or `catch (e) { return null }` with no logging, no rethrow, no recovery. Hides bugs.
- Severity: 3-4. `proposed_action: rewrite` if intent allows error logging; `delete` (the catch) if not.

### defensive-internal
`try/catch` or input validation at an **internal** boundary (your code calling your code). The framework or type system already guarantees the precondition.
- Severity: 2-3.

### duplicate-and-drift
Two or more functions/blocks doing the same thing with minor variations. Detect by AST-shape similarity or near-identical line sequences.
- Severity: 3-4. `proposed_action: extract` if both still needed, `delete` for the dead copy.

### god-file
Single file > 800 LOC OR exporting > 10 unrelated symbols. The file does too much.
- Severity: 3. `proposed_action: extract` (suggest the split).
- Skip if intent explicitly declares "single-file architecture".

## Detection technique

1. Build a quick map: `find <target> -type f -name '*.ts' -o -name '*.tsx' -o -name '*.js' -o -name '*.jsx' -o -name '*.vue' -o -name '*.py' -o -name '*.go' -o -name '*.rs'` (adapt to declared stack).
2. For each export, grep usages.
3. For control-flow, read the file and scan for the patterns above.
4. For duplicates, sample 50-line windows and cluster by hash/shape.

## Risk notes

Always state what could break:
- "Used only in tests — deleting kills 3 tests"
- "Imported via dynamic `require()` — static grep would miss; verify before burning"
- "Exported but only re-exported from a barrel; check if any external consumer imports the barrel"

## What you do NOT touch

- Anything matching `INTENT.md` forbidden anti-patterns (those are calcinate-friendly; you flag them).
- Anything serving the declared business intent — even if it looks ugly.
- Test files unless they're testing already-removed code.
- Generated files (look for `// AUTO-GENERATED`, `# GENERATED BY`, etc.).
